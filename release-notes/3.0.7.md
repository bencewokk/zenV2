Account status is now treated as the source of truth for paid access.

- The app now calls the authenticated account endpoint and treats `authenticated: false` as logged out, which denies paid access.
- When authenticated, Zen reads `user.subscription.status` as the primary access signal and shows the user type in Settings.
- The billing panel now displays `plan`, `currentPeriodEnd`, `stripeCustomerId`, and `stripeSubscriptionId` for debugging.
- Missing, unavailable, or malformed subscription data now falls back to no paid account.
- The release workflow can use `RELEASE_GITHUB_TOKEN` from GitHub variables or secrets, with the default Actions token as a fallback.
