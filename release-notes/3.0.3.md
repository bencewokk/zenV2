Zen now brings course material, research, cloud files, code, and web captures into one connected source library.

- Added a new **Sources** workspace with provider filters, full-text search, source previews, original links, citation copying, and automatic refresh.
- Added read-only **Canvas** support for courses, assignments, submission state, modules, announcements, and course files.
- Added selected-folder **Google Drive** indexing for Docs, Slides, Sheets, text files, and file metadata. Google authorization now requests the read-only Drive scope.
- Added **Zotero** libraries and collections with papers, annotations, authors, tags, DOI metadata, citation keys, and formatted citations.
- Added **GitHub** repository indexing for documentation, source files, issues, and pull requests, with configurable generated/vendor path exclusions.
- Added the unpacked **Zen Web Capture** browser extension for saving clean pages, selected passages, and visible-page screenshots into Sources.
- Connected sources now participate in AI context automatically. The assistant can search and read sources, refresh connections, cite sources into notes, and produce clickable source citations.
- Added a Google-owned encrypted connection vault for AI, Canvas, Zotero, and GitHub credentials. Existing credentials migrate automatically, restore on another device, can be revoked individually, and are cleared locally on Google sign-out.
- Redesigned onboarding around a Google-first Zen identity followed by optional, permission-conscious source connections.
- Connection settings and selections sync without leaking secrets; cached provider content remains local and can be rebuilt from its original service.

### Deployment note

The sync API must be deployed with a persistent 32-byte `CONNECTION_VAULT_KEY` environment variable before account-vault backup and restore can be tested. Changing that key makes existing encrypted vault records unreadable.
