The tool catalog was built at module-eval time and resolved ids through the
feature stores, but tools.ts sits in an import cycle with them (home/store →
ai/store → tools). Entering the cycle store-first — chunk-order dependent, hit
by the quiz/lesson surfaces — threw "Cannot access 'useHome' before
initialization" and black-screened the app. The catalog is now built lazily on
first access (toolCatalog()).

Crash handling added around it:
- index.html boot handler: paints a recovery screen with the error + reload
  when a module-eval crash fires before React ever mounts
- QuizView / LessonMode / ChatPanel each get an ErrorBoundary that closes the
  crashed surface (clearing its persisted state — no reload crash loop) and
  toasts, instead of unmounting the whole app
