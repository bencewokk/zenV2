The auto-updater never found new releases because tauri-action runs the
beforeBuildCommand (npm run build), not npm run tauri:build, so
sync-version.mjs never ran in CI. Every published build shipped the stale
tauri.conf.json version (0.2.3), so latest.json always reported 0.2.3 and
check() saw no newer version.

- Run scripts/sync-version.mjs inside beforeBuildCommand so the bundle and
  updater latest.json get the real tag version.
- fetch-depth: 0 in the release workflow so git describe --tags resolves.
- Sync the stale package.json/tauri.conf.json fallback versions.
