The v0.2.9 release feed reported version 0.2.9 but bundled installers named
0.2.8: tauri-action takes latest.json's version from the tag, while the
bundle uses tauri.conf.json, which sync-version.mjs had set from `git
describe` — and describe didn't resolve to the pushed tag in CI.

scripts/version.mjs now honours a ZEN_RELEASE_VERSION env override, set in
release.yml from github.ref_name. The bundle, frontend __APP_VERSION__, and
updater feed now all agree with the release tag.
